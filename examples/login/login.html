Login

The login process consists of the user entering a unique username and password pair.

hash = sha512(sha512(username) + password)

This allows the testing of userd by checking for the existence of this hash.

On the checks success — that the hash is present in userd's memory — the user can be safely logged in using the you're usual methods. In this capacity userd is used as a buffer mechanism between login and back-end user database.

Each check transaction will take at least one second, this is a feature of the client to prevent brute-force attacks on the login.
